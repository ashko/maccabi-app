# RideCoach — אפיון מערכת שיבוץ אימונים חכמה

מערכת ווב אוטומטית לבניית לו״ז אימונים שבועי למאמן כושר ותזונה, כולל
אופטימיזציית מסלול על אופניים ושליחת הודעות וואטסאפ אישיות.

> תצוגה חזותית מלאה של האפיון זמינה כ-Artifact (מסמך RTL אינטראקטיבי).

---

## 1. פירוק הבעיה

חמש הדרישות מהקייס מתפרקות לארבעה אתגרים הנדסיים:

| # | אתגר | מהות |
|---|------|------|
| 1 | שני סוגי שיבוץ | **קבועים** (יום+שעה נעולים, חוזרים) כעוגנים; **משתנים** שיש לשבץ כל שישי בתוך חלונות זמינות |
| 2 | שני תחומי אימון | **גופני** (פרונטלי); **תזונה** (פרונטלי או מרחוק — מרחוק לא מוסיף נסיעה) |
| 3 | ניתוב אופניים | מזעור **זמן רכיבה כולל** מהבית וחזרה, בפרופיל אופניים (לא רכב) |
| 4 | הודעות אישיות | וואטסאפ מותאם אישית לכל מתאמן אחרי אישור הלו״ז |

**התובנה המרכזית:** זו בעיית **VRPTW** (Vehicle Routing Problem with Time
Windows) עם רכב יחיד (המאמן), אילוצי עוגן (הפגישות הקבועות) וחלונות זמן
(זמינות מתאמנים + שעות עבודה). את כל השבוע פותרים במודל אחד.

---

## 2. משתמשים

- **המאמן** — מגדיר בית/שעות/מרווחים, מנהל מתאמנים ואימונים קבועים, מריץ
  אופטימיזציה בשישי, בוחן על מפה+יומן, מכוונן, מאשר ושולח.
- **המתאמן** — מקבל הודעת וואטסאפ אישית; מעדכן זמינות דרך קישור אישי; מאשר/מבקש שינוי.
- **המערכת** — מזכירה בשישי, בונה מטריצת זמנים, מריצה סולבר, שולחת ועוקבת.

---

## 3. טקס יום שישי (מקצה לקצה)

1. **איסוף נתונים** (Fri 07:00, אוטומטי) — קבועים + בקשות משתנות + חלונות זמינות.
2. **גיאוקוד + מטריצה** (אוטומטי) — כתובות→קואורדינטות, בניית מטריצת זמני רכיבה.
3. **אופטימיזציה** (אוטומטי, ~שניות) — OR-Tools מייצר לו״ז: יום, שעה, סדר מסלול.
4. **בחינה וכיוונון** (מאמן) — יומן+מפה, גרירה/נעילה/הרצה-מחדש.
5. **אישור** (מאמן) — לחיצה מקבעת את הלו״ז.
6. **שליחת וואטסאפ** (אוטומטי) — הודעה אישית + מעקב מסירה/קריאה/תגובות.

## 4. מנוע השיבוץ (OR-Tools)

**מידול חכם — כל יום בשבוע = "רכב" נפרד ב-VRP:**

- **מוסך (Depot):** בית המאמן — התחלה וסיום של כל יום.
- **צמתים:** כל פגישה = צומת לביקור במהלך השבוע.
- **רכבים = ימים:** 6 רכבים (א׳–שישי), לכל אחד חלון זמן = שעות העבודה של אותו יום.
- **עוגנים קבועים:** ננעלים ליום ולשעה (חלון זמן ברוחב ~0).
- **פגישות משתנות:** חופשיות לכל יום/שעה בתוך חלונות הזמינות.
- **תזונה מרחוק:** "ללא מיקום" — צורכת זמן ביומן אך לא מוסיפה נסיעה.

**פונקציית מטרה:** מזעור זמן הרכיבה הכולל לאורך השבוע.

**אילוצים קשיחים:** חלונות זמינות · שעות עבודה · משך אימון · מרווח מנוחה ·
מקס׳ פגישות ליום · עוגנים קבועים.
**אילוצים רכים (עונשים):** העדפות שעה · איזון עומס בין ימים · קיבוץ גאוגרפי.

**אי-כדאיות (Infeasible):** אם אין פתרון — הצגת הקונפליקט המדויק למאמן והצעת
הרפיית אילוץ, במקום כישלון שקט.

*למה OR-Tools:* פותר הקצאה-ליום + סידור-מסלול בו-זמנית, קוד-פתוח, שניות בסקאלה
של עשרות מתאמנים.

## 5. שכבת ניתוב גאוגרפי

- **גיאוקודינג** — כתובות→קואורדינטות פעם אחת + מטמון (OpenRouteService / Nominatim / Google).
- **מטריצת זמני רכיבה** — N×N בפרופיל **cycling** (שבילי אופניים + שיפועים), לא רכב
  (ORS Matrix / GraphHopper / OSRM bike).
- **שיקול ישראל** — כיווני אופניים ב-Google מוגבלים; ברירת מחדל **OpenRouteService**
  / GraphHopper (ניתן לארח עצמאית).
- **מטמון** — המטריצה משתנה לאט; מחשבים פעם אחת ומרעננים דלתות בלבד.

---

## 6. שכבת הודעות (WhatsApp)

שליחה יזומה מחייבת **WhatsApp Cloud API** עם **תבנית מאושרת** (Utility) ומשתנים:

```
היי {{1}}, הנה האימון שלך לשבוע הבא: {{2}} ביום {{3}} בשעה {{4}} — {{5}}. נתראה! 🚴
   (1=שם, 2=סוג, 3=יום, 4=שעה, 5=מיקום)
```

**אמינות ותאימות:**
- **Opt-in** — תיעוד הסכמת מתאמן לפני שליחה.
- **שליחה אידמפוטנטית** — מפתח ייחודי לכל הודעה, אפס כפילויות גם בריטריי.
- **מעקב מצב** — sent → delivered → read → failed, עם ריטריי ותור מכתבים-מתים.
- **חלון 24 שעות** — תגובת מתאמן פותחת חלון שיחה לתיאום שינויים.

ספקים: Meta Cloud API (ישיר, זול) · Twilio / 360dialog (BSP). אופציונלי: קישור יומן `.ics`.

## 7. מודל נתונים (8 ישויות ליבה)

ההפרדה בין **תבנית** (קבוע/בקשה) ל**מופע** (פגישה משובצת) מאפשרת ייצור לו״ז חדש כל שבוע.

| ישות | שדות מרכזיים |
|------|--------------|
| `Trainer` | home_geo, work_hours[], buffer_min, max_per_day, bike_profile |
| `Trainee` | name, phone, address→geo, consent, notes |
| `SessionType` | category(phys/nutri), duration, is_remote |
| `FixedSession` | trainee, type, weekday, start_time, location |
| `SessionRequest` | trainee, type, frequency, availability[] |
| `Availability` | trainee, weekday, from, to |
| `ScheduledSession` | date, start, end, order, day_idx, status |
| `WeeklyPlan` | week_start, status, total_ride_time, approved_at |
| `MessageLog` | session, template, wa_status, sent/read_at |

**סטטוס פגישה:** `proposed → approved → sent → confirmed → done / canceled`.

## 8. ארכיטקטורה

```
Client:   Next.js/React — יומן + מפה אינטראקטיבית + קישור זמינות ציבורי
   ↕
API:      FastAPI (Python) — אימות + אורקסטרציה + תור (Celery/Redis)
   ↕
Services: OR-Tools (אופטימיזציה) · OpenRouteService (גאו+מטריצה)
          WhatsApp Cloud API (הודעות) · Cron (טריגר שישי 07:00)
   ↕
Data:     PostgreSQL + PostGIS (גאו+יחסי) · Redis (תור + מטמון מטריצה)
```

- **Frontend:** Next.js · React · Leaflet/MapLibre (מפה) · FullCalendar (יומן).
- **Backend:** Python/FastAPI — סינרגיה עם OR-Tools וספריות גאו.
- **Jobs:** Celery + Redis / Cloud Scheduler.
- **Hosting:** Frontend על Netlify/Vercel · Backend+workers על Railway/Fly/VPS.

## 9. תוכנית מימוש בשלבים

- **MVP (~4 שבועות)** — ניהול מתאמנים/אימונים · ניתוב חמדני (nearest-neighbor) עם
  מטריצת אופניים · תצוגת לו״ז · אישור ידני · שליחת וואטסאפ בתבנית מאושרת.
  מוכיח את השרשרת המלאה.
- **V2 — אופטימיזציה** — OR-Tools מלא (חלונות זמן, איזון עומס) · מפה אינטראקטיבית
  (גרירה/נעילה/הרצה-מחדש) · קישור זמינות עצמי למתאמן.
- **V3 — אוטומציה מלאה** — אופטימיזציה-מחדש בביטול/שינוי · לוח בקרת מסירות ·
  אנליטיקות · הרחבה מולטי-מאמן (SaaS).

## 10. סיכונים, פרטיות ועלויות

- **פרטיות** — PII (שם/טלפון/מיקום): הצפנה במנוחה, מזעור נתונים, תיעוד הסכמה —
  בהתאם לחוק הגנת הפרטיות ולתנאי WhatsApp.
- **אי-כדאיות** — הצגת קונפליקט מדויק + הצעת הרפיה, לא כישלון שקט.
- **עלויות** — וואטסאפ לפי שיחה (Utility) · ניתוב ORS בחינם/אירוח-עצמי · אירוח צנוע.

**שני עקרונות מצילים:** (1) שליחה **אידמפוטנטית** — לעולם לא פעמיים.
(2) כל טעינת אילוץ מייצרת **לוג קונפליקט קריא** — שהמאמן תמיד יבין את השיבוץ.
